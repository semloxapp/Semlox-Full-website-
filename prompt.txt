# BUG 1 — INPUT FOCUS LOST AFTER TYPING ONE CHARACTER

## Current behavior

When I click into an AWB Form field and type one character:

- the character is entered
- immediately the input loses focus
- I have to click the field again to type another character

This is a serious controlled-input/rendering bug.

## Required investigation

Inspect the AWB Form field rendering.

Look specifically for:

- unstable React keys
- keys based on field value
- keys based on edited value
- keys based on JSON.stringify
- keys using Date.now()
- keys using Math.random()
- conditional component remounting
- map rendering where key changes after each input update
- switching between textarea/input based on value
- parent component remounting on each keystroke
- state update rebuilding the entire AWB form tree
- active tab/state causing full remount
- derived presentation data recreated in a way that changes keys

Search for:
```txt
key=
Math.random
Date.now
JSON.stringify
onChange
setState
setData
setEdited
setFields
AwbDocumentPreview
AwbPaperContent
renderField
textarea
input
contentEditable
````

## Required fix

The input must keep focus while typing.

Rules:

* React keys for AWB form fields must be stable.
* Use stable field id/name/path as key.
* Do NOT use field value as key.
* Do NOT use edited value as key.
* Do NOT remount the whole AWB form on every keystroke.
* State updates should only update the relevant field value.
* The textarea/input element should remain the same DOM node while editing.

## Expected result

After fix:

* Click any AWB Form field.
* Type multiple characters continuously.
* Focus stays inside the field.
* Cursor does not jump.
* No need to click again after each character.
